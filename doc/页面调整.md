# 页面调整

## 调整背景

本次调整针对博客详情页（例如 `getting-started`）右侧目录导航和正文阅读体验进行了优化，主要解决以下几个问题：

1. 博客正文与标题字号偏小，阅读时显得紧凑。
2. 右侧目录导航顶部文案和目录项字号偏小，与正文不够协调。
3. 右侧顶部文案默认态和激活态过暗，辨识度不够。
4. 右侧顶部文案从 `Contents` 改为更简洁的 `TOC`。

## 本次调整内容

### 1. 调整博客正文与标题字号

- 在全局样式中新增变量 `--blog-reading-size: 1.0625rem` 作为博客阅读字号基准。
- 将 `.prose` 的基础字号从 `1rem` 调整为 `var(--blog-reading-size)`。
- 由于文章标题层级使用的是相对 `em` 单位，基础字号抬高后，标题和正文会一起按比例变大，整体阅读感受更舒适。

### 2. 调整右侧 TOC 字号

- 将右侧目录触发文案的字号改为 `var(--blog-reading-size)`。
- 将右侧目录列表容器字号也改为 `var(--blog-reading-size)`。
- 这样可以保证 `TOC` 顶部文案、目录列表、文章正文三者的字号基准一致。

### 3. 调整右侧顶部文案亮度

- 将右侧顶部触发文案颜色从固定的灰色改为 `var(--c-text)`，使其跟随当前主题颜色。
- 提高默认态透明度，使未激活时也更容易看清。
- 提高悬停态和激活态透明度，让交互反馈更明显。
- 目录列表文字亮度未做额外调整，保持原有层次。

### 4. 文案调整

- 右侧目录顶部文案由 `Contents` 改为 `TOC`。

## 涉及文件

- `src/styles/main.css`
  - 新增博客阅读字号变量 `--blog-reading-size`
- `src/styles/prose.css`
  - 调整博客正文基础字号
- `src/components/base/DesktopAside.astro`
  - 调整右侧顶部文案的字号、颜色、透明度
  - 将 `Contents` 改为 `TOC`
- `src/components/toc/TocSidebar.astro`
  - 调整右侧目录列表字号

## 当前效果

- 博客正文和标题比之前更大，阅读体验更自然。
- 右侧 `TOC` 标题和目录项字号与正文更加协调。
- `TOC` 顶部文案在默认态和激活态下都更清晰。
- 整体保持原有页面结构和交互方式不变，只优化了视觉呈现。
