# `wutong-yu` 项目解析

> 本文基于当前磁盘中的真实代码整理，目标是把这个项目“现在有哪些功能、每个代码文件负责什么、数据怎么流动、当前有哪些已知问题”完整讲清楚。

---

## 1 项目定位

这是一个基于 **Astro 5** 的精简个人站点，风格来自 `astro-antfustyle-theme`，但已经收缩成一个更聚焦的内容站。

### 当前保留的核心功能

- 首页 `/`
- 博客列表 `/blog/`
- 博客详情 `/blog/[...slug]/`
- 项目页 `/projects/`
- Pagefind 搜索
- 明暗主题切换
- 文章 TOC 目录
- 三种背景动画 `plum / dot / rose`
- OG 图片自动生成

### 页面与数据源对应关系

| 页面 | 路由入口 | 数据来源 | 核心视图组件 |
| :--- | :--- | :--- | :--- |
| 首页 | `src/pages/index.mdx` | `src/content/home/index.md` | `RenderPage.astro` |
| 博客列表 | `src/pages/blog/index.mdx` | `src/content/blog/*.md` | `ListView.astro` |
| 博客详情 | `src/pages/blog/[...slug].astro` | `src/content/blog/*.md` | `RenderPost.astro` |
| 项目页 | `src/pages/projects.mdx` | `src/content/projects/data.json` | `GroupView.astro` |
| 404 | `src/pages/404.mdx` | frontmatter | `StandardLayout.astro` |

### 整体运行链路

```text
src/content/* 写内容
        ↓
content.config.ts 定义集合
        ↓
schema.ts 做结构校验
        ↓
pages/ 决定路由入口
        ↓
layouts/ 负责页面骨架
        ↓
views/ 把内容渲染成页面
        ↓
styles/ + UnoCSS 负责最终视觉
        ↓
plugins/ 在 Markdown 渲染阶段补充阅读时长、OG 图、标题锚点等能力
```

## 4 六个核心配置文件

### 4.1 `astro.config.ts`

这是 Astro 总配置，控制：

- 站点地址 `site`
- 子路径部署 `base`
- `sitemap()`、`robotsTxt()`、`unocss()`、`astroExpressiveCode()`、`mdx()`
- Markdown 插件链
- 图片优化配置
- Astro 实验项

### 4.2 `ec.config.mjs`

这是代码块展示配置，控制：

- Expressive Code 的主题
- 行号
- 折叠代码块
- 终端 / 编辑器框样式

### 4.3 `unocss.config.ts`

负责：

- 开启 `presetWind3`、`presetAttributify`、`presetIcons`、`presetWebFonts`
- 注册 `transformerDirectives` 和 `transformerVariantGroup`
- 定义 `btn-*`、`*-transition`、`shadow-custom_*` 等 shortcut
- 通过 `safelist` 保证动态类名不会被漏掉

### 4.4 `tsconfig.json`

主要作用：

- 继承 Astro 严格模式
- 定义路径别名 `~/* -> ./src/*`

### 4.5 `eslint.config.js`

主要作用：

- 启用 JS、TS、Astro、a11y 规则
- 合并 `globals.browser` 和 `globals.node`
- 对未使用变量做严格约束
- 与 Prettier 兼容

### 4.6 `package.json`

它既是依赖清单，也是整个工程的命令入口。

其中 `markdown-syntax-guide.md` 的意义很大，它其实是"样式和渲染回归测试样本"。

### 博客文章图片管理方案

推荐使用 `src/assets/` 目录存储博客图片，可享受 Astro 自动优化（压缩、WebP 转换、响应式）。

**目录结构：**

```text
src/
├── assets/
│   ├── post-1/
│   │   ├── img1.jpg
│   │   └── img2.jpg
│   └── post-2/
│       ├── img1.jpg
│       └── img2.jpg
└── content/
    └── blog/
        ├── post-1.md
        └── post-2.md
```

**Markdown 中引用方式：**

```markdown
<!-- 相对路径 -->
![图片描述](../../assets/post-1/img1.jpg)

<!-- 别名路径 -->
![图片描述](@/assets/post-1/img1.jpg)
```

> 相比 `public/` 目录方案，`src/assets/` 方案会让 Astro 在构建时自动优化图片，生成哈希文件名并输出到 `dist/_astro/`，对性能更友好。

## 17 实际检查结果

### `pnpm check`

结果：**通过**

- 0 errors
- 0 warnings
- 0 hints

但是内容同步阶段出现了一个关键告警：

```text
Duplicate id "index" found in src/content/home/index.md
```

根因是：

- `content.config.ts` 定义了 `pages` collection，扫描 `src/pages/**/*.mdx`
- `src/pages/index.mdx` 的 id 是 `index`
- `src/content/home/index.md` 的 id 也是 `index`

### `pnpm build`

构建 **没有完全成功**，最终报错：

```text
ENOENT: no such file or directory, lstat 'dist/chunks/StandardLayout_*.mjs'
```

这说明当前项目还不属于“完整可交付”的稳定状态。

### 额外观察到的构建信息

`Dot.astro` 的客户端产物体积很大：

```text
dist/_astro/Dot....js  1068.28 kB
gzip: 269.88 kB
```

这意味着 `p5` 背景会明显抬高前端包体积。

## 18 文档末尾问题记录（后续修复清单）

### 一、构建与配置问题

1. `src/config.ts` 里的 `SITE.website` 仍然是 `https://example.com/`
2. `pnpm build` 当前失败，报 `dist/chunks/StandardLayout_*.mjs` 缺失
3. `src/content.config.ts` 中的 `pages` collection 已建模，但当前代码没有真正消费它
4. `src/pages/index.mdx` 与 `src/content/home/index.md` 产生重复 `index` ID 告警
5. `src/config.ts` 中的 GitHub 社交链接仍指向上游主题仓库

### 二、逻辑与行为不一致

6. `schema.ts` 说明 `ogImage: false` 可禁用 OG 图，但 `Head.astro` 仍会回退到默认图
7. `src/components/toc/Toc.astro` 注册自定义元素时没有做防重复保护
8. `src/components/backgrounds/Plum.astro` 同样没有做防重复注册保护
9. `NavSwitch.astro` 只负责打开面板，没有同步 `aria-expanded`
10. `SearchSwitch.astro` 注释写的是“open or close”，但当前按钮逻辑只有打开

### 三、可访问性与文案问题

11. `src/components/views/GroupItem.astro` 给整张外链卡片写了 `aria-hidden={true}`，会让读屏器忽略该链接
12. `src/pages/404.mdx` 仍是模板式占位文案，正文只有 `Nice to meet you tho!`
13. `src/components/views/GroupView.astro` 的空态提示仍是面向开发者的模板文案
14. `src/content/home/index.md` 中仍保留了指向上游主题仓库的内容说明

### 四、性能与维护成本问题

15. `Dot.astro` 包体积过大，弱网和首屏表现都会受影响
16. `markdown.css` 里保留了不少当前站点暂未实际使用的模板能力样式
17. `schema.ts` 中 `radio`、`video`、`platform`、`redirect` 等字段当前没有真实内容在用

## 20 一句话总结

这个项目已经完成了“能写内容、能展示博客、能展示项目、能搜索、能切换主题”的主体搭建，但它现在仍处于“从通用主题裁剪到个人站点的过渡期”。
